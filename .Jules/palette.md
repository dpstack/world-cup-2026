## 2024-04-10 - Accessible Empty States & Search Input UX
**Learning:** Adding custom clear buttons on search inputs can introduce a double-clear-button glitch in Webkit browsers when `type="search"` is used. Using `type="text"` with `role="searchbox"` prevents this native browser collision while retaining semantic accessibility. Empty states with proactive DOM insertion of `role="status"` improve screen-reader notifications, especially for highly dynamic client-side filtering arrays like `CountriesGallery.jsx` where visual cues might not trigger readouts for blind users.
**Action:** When implementing custom search functionality, default to `<input type="text" role="searchbox">` if planning to render custom clear actions (`x`), and actively wrap empty filtered array states with `role="status"` for immediate verbal feedback.
