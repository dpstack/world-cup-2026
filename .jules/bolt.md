## 2024-05-24 - Memoize `computeTable` array processing with WeakMap
**Learning:** Found a major performance bottleneck where `getLiveStandings` called `computeTable` on 12 groups for every single score update across the app. Because the groups' matches arrays in React state are largely unchanged across those re-renders, the immutable `matches` arrays make great keys for a `WeakMap`. However, additional complex/default params (`teams`, `tieBreakers`) could cause cache misses or leak memory on defaults, so stringifying them inside the WeakMap value acts as a safe composite key pattern.
**Action:** Always check React components updating array/object state for complex dependent computations. If state slices are untouched, caching calculations keyed directly on those untouched array/object references with `WeakMap` yields huge O(1) skips for expensive operations.
